# PCI-DSS Mini Compliance Pack — Additional Files

This folder contains additional supporting files that complement the core PCI-DSS Compliance Pack.  
These files help demonstrate your understanding of evidence management, control mapping, and audit preparation in a real compliance workflow.

---

## 📁 Contents

### ✅ Evidence Placeholder Files
These files simulate what real audit evidence looks like.  
In a real PCI-DSS engagement, auditors expect to see tickets, logs, reports, and configuration reviews.

- **Sample_Firewall_Rule_Review.txt**  
  Placeholder representing a firewall rule review artifact.

- **Sample_Log_Review_Report.txt**  
  Placeholder representing a monthly or weekly log review summary.

- **Sample_ASV_Scan_Summary.txt**  
  Placeholder representing quarterly external vulnerability scan results (ASV).

- **Sample_Audit_Ticket.txt**  
  Placeholder representing an audit or remediation ticket used to track compliance activities.

These placeholders will be replaced with more realistic mock evidence or screenshots later in the training program.

---

### ✅ PCI-ISO-NIST Mapping (Cross-Framework Alignment)
- **PCI_ISO_NIST_Mapping.csv**

This file provides a simplified crosswalk between:
- PCI-DSS Requirements  
- ISO/IEC 27001 Annex A controls  
- NIST Cybersecurity Framework (CSF) functions/categories  

Cross-framework understanding is essential for compliance and GRC professionals working in multi-standard environments.
