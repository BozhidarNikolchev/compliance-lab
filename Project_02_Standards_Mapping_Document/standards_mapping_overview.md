# Standards Mapping Overview

This document provides an overview of how core security & privacy requirements align across:

- ISO 27001:2022  
- NIST CSF 2.0  
- SOC 2  
- PCI-DSS v4.0  
- GDPR  

## Purpose

Most frameworks overlap heavily. By mapping them:

- We eliminate duplicate work during audits  
- We unify internal controls  
- We reuse evidence across multiple certifications  
- We establish a single source of truth for governance  

## Scope

Included categories:

- Access Control  
- Asset Management  
- Logging & Monitoring  
- Incident Response  
- Risk Management  
- Data Protection / GDPR compliance  
- Vendor Management  

## Output Files

- `control_mapping_matrix.csv` — main control mapping  
- `evidence_alignment_table.md` — evidence-to-control mapping  
- `mapping_process_steps.md` — methodology
