# Security Audit Procedure

## Phase 1: Planning  
- Define scope  
- Identify controls in scope  
- Establish criteria  

## Phase 2: Document Review  
- Review policies, standards, procedures  
- Review previous audits  
- Review risk register  

## Phase 3: Interviews  
Interview: IT, HR, Security, Management, DPO.

## Phase 4: Testing  
- Check access provisioning  
- Verify logging configuration  
- Review backups  
- Inspect vendor contracts  
- Validate evidence  

## Phase 5: Findings  
Document:
- Issue  
- Evidence  
- Risk  
- Recommendation  

## Phase 6: Reporting  
Write final report using `report_template.md`.
